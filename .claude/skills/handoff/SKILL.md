# Handoff — ส่งต่องาน PD Monitor

ทำตามขั้นตอนนี้ให้ครบและเรียงลำดับ เพื่อให้ session หน้ากลับมาทำต่อได้ทันทีโดยไม่ต้องเดา

## 1. อัปเดต `note.md` (ที่ราก)
- เพิ่มบล็อกบันทึกใหม่ลงท้าย ขึ้นหัวด้วยวันที่ (พ.ศ.) เช่น `## 📅 บันทึกวันที่ DD เดือน 25xx`
- เนื้อหา: **วันนี้ทำอะไรไป** (bullet), **ไฟล์/ส่วนที่เปลี่ยน**, **▶️ พรุ่งนี้เริ่มตรงนี้ (next steps)**
- อัปเดตหัวข้อ **"📌 สถานะปัจจุบัน"** ท้ายไฟล์ให้ตรงความจริงล่าสุด (เฟสไหน เสร็จอะไรแล้ว)
- เขียนภาษาไทย กระชับ ให้คนไม่ใช่โปรแกรมเมอร์อ่านรู้เรื่อง

## 2. อัปเดต memory
- แก้ไฟล์ `C:\Users\mkt01\.claude\projects\C--Users-mkt01-Desktop-web-app\memory\pd-monitor-project.md`
  ให้สถานะตรงปัจจุบัน (เฟส/สิ่งที่ตัดสินใจใหม่) — แปลงวันที่สัมพัทธ์เป็นวันที่จริงเสมอ
- ถ้ามี memory ใหม่ที่ควรแยกไฟล์ ให้สร้างแล้วเพิ่มบรรทัดใน `MEMORY.md`
- อย่าบันทึกสิ่งที่โค้ด/git บอกอยู่แล้ว — เก็บเฉพาะบริบทที่เดาไม่ได้

## 3. Commit + push (Vercel auto-deploy)
```bash
cd "C:\Users\mkt01\Desktop\web app"
git add -A
git status            # ยืนยัน .env.local ไม่หลุดขึ้น
git commit -m "<สรุปงานสั้นๆ>"
git push
```
- เช็กก่อน commit ว่า `.env.local` ไม่อยู่ใน staged (ถ้าหลุด → หยุดแล้วแจ้งผู้ใช้)

## 4. (ออปชัน — ถามผู้ใช้ก่อนเสมอ) โพสต์ความคืบหน้ากลับ Notion
- ถาม: "อยากให้โพสต์สรุปงานวันนี้กลับไปที่หน้า Notion ของทีมไหม?"
- ถ้าตกลง: เพิ่มบันทึกสั้นๆ (วันที่ + ทำอะไรไป) ใต้หน้าโปรเจคใน Notion
  (หน้าหลัก `38092ef2-c18f-803d-bb95-f49546a4f466`) ผ่าน Notion MCP
- ⚠️ นี่คือการแก้ workspace ของทีม → ห้ามทำเองโดยไม่ได้รับอนุญาต
