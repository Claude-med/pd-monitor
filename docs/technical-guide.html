01 · ทำความเข้าใจ

โปรเจคนี้เราจะทำอะไร

เราจะสร้าง เว็บแอปพลิเคชัน ให้โรงงานยาใช้บันทึกและติดตามงานผลิต แทนการจดลงกระดาษ โดยทุกฝ่ายเข้ามาบันทึก/ดูข้อมูลผ่านเว็บเดียวกัน (เปิดบนคอมหรือมือถือก็ได้) ข้อมูลเก็บรวมศูนย์บนคลาวด์

🎯 สิ่งที่ต้องสร้าง (ฟีเจอร์หลัก)

  • บอร์ดติดตามคำสั่งผลิต (Pending Order Board)
  • ฟอร์มบันทึกการผลิตแต่ละขั้น (เข้า-ออก, เครื่อง, คน, เวลา)
  • รายงานประจำวัน (Daily Report)
  • แดชบอร์ด KPI สำหรับผู้บริหาร
  • ระบบล็อกอิน + แยกสิทธิ์ตามฝ่าย

🧩 แนวคิดง่าย ๆ ของระบบ

เว็บแอป = 3 ส่วนทำงานร่วมกัน
1) หน้าเว็บ (Frontend) ที่ผู้ใช้เห็นและกด
2) ฐานข้อมูล (Database) ที่เก็บข้อมูลทั้งหมด
3) ตัวกลาง (Backend/API) ที่รับส่งข้อมูลระหว่างสองส่วนแรก
โชคดีที่ stack ที่เราเลือกรวมส่วนเหล่านี้ให้แทบครบในที่เดียว ทำให้ทีมเล็กทำได้

🔄 ภาพการทำงานของระบบ (Data Flow)

# ผู้ใช้กดบันทึกงาน → ข้อมูลวิ่งไปเก็บ → ทุกคนเห็นพร้อมกัน [ ผู้ใช้: ฝ่ายผลิต/QC/QA ] │ กรอกฟอร์มบนเว็บ (มือถือ/คอม) ▼ [ Next.js หน้าเว็บ + API ] ── โฮสต์บน Vercel │ ส่งคำสั่งบันทึก/ดึงข้อมูล ▼ [ Supabase = ฐานข้อมูล PostgreSQL + Auth ] ── คลาวด์ │ อัปเดตเรียลไทม์ ▼ [ ทุกฝ่ายเห็นสถานะงานล่าสุดตรงกัน ]

💡 ข่าวดีสำหรับทีม

เราใช้ Claude Code (AI ช่วยเขียนโค้ด) เป็นผู้ช่วยหลัก น้อง ๆ ไม่จำเป็นต้องเก่งโปรแกรมมาก่อน หน้าที่หลักคือ เข้าใจภาพรวม สั่งงานให้ชัด ทดสอบผล และตัดสินใจ ส่วนการพิมพ์โค้ดให้ AI ช่วย แล้วเราตรวจทาน

02 · เทคโนโลยีที่ใช้

Tech Stack & ภาษาที่ใช้เขียน

นี่คือ “เครื่องมือทั้งชุด” ของโปรเจค พร้อมเหตุผลว่าทำไมถึงเลือก:

📘
TypeScript — ภาษาหลักของโปรเจค
ภาษาหลักที่ใช้เขียนทั้งหน้าเว็บและตรรกะ เป็น JavaScript ที่เพิ่ม “ชนิดข้อมูล” ช่วยกันพิมพ์ผิดและจับ bug ได้ตั้งแต่ตอนเขียน
.ts / .tsx
⚛️
Next.js (ใช้ React) — เฟรมเวิร์กหน้าเว็บ
สร้างหน้าเว็บและ API ในตัวเดียว ทำเว็บได้ทั้งเร็วและรองรับมือถือ เป็นมาตรฐานที่นิยมและเข้าได้กับ Vercel โดยตรง
React / JSX
🎨
Tailwind CSS + shadcn/ui — งานหน้าตา (UI)
จัดรูปแบบหน้าเว็บด้วยคลาสสำเร็จรูป ทำให้สวยและเหมือนกันทั้งระบบเร็ว ส่วน shadcn/ui คือชุดปุ่ม/ตาราง/ฟอร์มสำเร็จรูปพร้อมใช้
CSS
🗄️
Supabase — ฐานข้อมูล + ระบบผู้ใช้
ให้ฐานข้อมูล PostgreSQL, ระบบล็อกอิน, การอัปเดตเรียลไทม์ และระบบสิทธิ์ (RLS) ครบในที่เดียว เขียนน้อยแต่ได้เยอะ
SQL
▲
Vercel — ที่โฮสต์เว็บ (Deploy)
เอาเว็บขึ้นออนไลน์ให้คนเข้าได้ เชื่อมกับ GitHub แล้วทุกครั้งที่ push โค้ด เว็บจะอัปเดตอัตโนมัติ
—
🐙
Git + GitHub — เก็บโค้ด/ทำงานเป็นทีม
เก็บประวัติโค้ดทุกเวอร์ชัน ย้อนกลับได้ และให้หลายคนทำงานพร้อมกันโดยไม่ทับกัน
CLI
🤖
Claude Code — AI ผู้ช่วยเขียนโค้ด
สั่งเป็นภาษาไทยได้ ช่วยเขียน/แก้/อธิบายโค้ดในเครื่องเราโดยตรง เป็นผู้ช่วยหลักของทีม
CLI

🔤 สรุป: ในโปรเจคนี้เราเขียนภาษาอะไรบ้าง

ภาษาใช้ทำส่วนไหนหมายเหตุ

03 · เตรียมตัว

สิ่งที่ต้องติดตั้ง & บัญชีที่ต้องสมัคร

รายการประเภททำไมต้องมี
Node.js (เวอร์ชัน LTS)โปรแกรมในเครื่องตัวรันโปรเจค JavaScript/TypeScript และเครื่องมือต่าง ๆ
VS Codeโปรแกรมในเครื่องโปรแกรมแก้ไขโค้ด (Editor) ที่นิยมที่สุด
Gitโปรแกรมในเครื่องเก็บประวัติโค้ดและเชื่อมกับ GitHub
Claude Codeโปรแกรมในเครื่องAI ผู้ช่วยเขียนโค้ด (ติดตั้งผ่าน npm)
บัญชี GitHubสมัครออนไลน์ (ฟรี)เก็บโค้ดบนคลาวด์ + เชื่อม Vercel
บัญชี Supabaseสมัครออนไลน์ (ฟรี)สร้างฐานข้อมูลของโปรเจค
บัญชี Vercelสมัครออนไลน์ (ฟรี)เอาเว็บขึ้นออนไลน์

⌨️ คำสั่งเริ่มต้น (รันใน Terminal ของ VS Code)

# 1) สร้างโปรเจค Next.js + TypeScript + Tailwind (ตอบ Yes ตามค่าแนะนำ) npx create-next-app@latest pd-monitor # 2) เข้าไปในโฟลเดอร์โปรเจค cd pd-monitor # 3) ติดตั้งตัวเชื่อมฐานข้อมูล Supabase npm install @supabase/supabase-js # 4) ติดตั้งชุด UI สำเร็จรูป shadcn/ui npx shadcn@latest init # 5) ลองรันเว็บในเครื่อง → เปิด http://localhost:3000 npm run dev

🔑 ไฟล์ลับ .env.local (ห้าม push ขึ้น GitHub)

เก็บ “กุญแจ” เชื่อม Supabase ไว้ในไฟล์นี้ และต้องอยู่ใน .gitignore เสมอ:

NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co NEXT_PUBLIC_SUPABASE_ANON_KEY=eyxxxx...

💡 จริง ๆ แล้วทุกขั้นตอนข้างบนนี้สั่ง Claude Code ทำให้ได้ทั้งหมด เช่นพิมพ์ว่า “ช่วยตั้งโปรเจค Next.js + Supabase + Tailwind ให้หน่อย” แล้วตรวจผลตาม

04 · โครงสร้าง

โครงสร้างไฟล์ในโปรเจค

pd-monitor/ ├── app/ # หน้าเว็บแต่ละหน้า (Next.js App Router) │ ├── board/ # หน้า Pending Order Board │ ├── job/[id]/ # หน้ารายละเอียด Job │ ├── record/ # หน้าบันทึกการผลิต │ ├── daily/ # หน้า Daily Report │ ├── dashboard/ # หน้า Dashboard/KPI │ └── layout.tsx # โครงร่วม (เมนู/หัวเว็บ) ├── components/ # ชิ้นส่วน UI ที่ใช้ซ้ำ (ปุ่ม, การ์ด, ตาราง) ├── lib/ │ └── supabase.ts # ตัวเชื่อมฐานข้อมูล ├── .env.local # กุญแจลับ (ไม่ push) ├── package.json # รายการ library ที่ใช้ └── tailwind.config.ts # ตั้งค่าธีม/สี

แต่ละโฟลเดอร์ใน app/ = หนึ่งหน้าเว็บ ทำให้หาง่ายและเพิ่มหน้าใหม่ได้สะดวก

05 · ฐานข้อมูล

ตารางในฐานข้อมูล (Database Schema)

ข้อมูลทั้งหมดเก็บเป็น “ตาราง” ใน PostgreSQL ที่เชื่อมโยงกัน:

ตารางเก็บอะไร
productsข้อมูลยา: ชื่อ, สูตร, batch size, รูปแบบบรรจุ, ขนาด, STD time
machinesเครื่องจักรแต่ละตัว และอยู่สถานีไหน
users / rolesผู้ใช้ และฝ่าย (ผลิต / QC / QA / คลัง / บัญชี / ผู้บริหาร)
ordersคำสั่งซื้อจากลูกค้า
jobsงานผลิต: เลข job, ยา, lot, batch size, สถานะ, แผนเดือน
job_stagesแต่ละขั้นของ job (ผสม/ตอก/บรรจุ/QC/QA/FG) + สถานะ
production_recordsบันทึกผลิต: เครื่อง, จำนวนเข้า-ออก, จำนวนคน, ชั่วโมง, เวลา
daily_reportsline clearance, set-up time, ETF ของแต่ละสถานี

🔗 ความสัมพันธ์: orders → มีหลาย jobs → แต่ละ job มีหลาย job_stages → แต่ละ stage มีหลาย production_records

07 · ทำงานเป็นทีม

ข้อตกลงการทำงาน (Conventions)

🐙 Git / GitHub

  • โค้ดหลักอยู่ที่ branch main — ห้าม commit ทับตรง ๆ
  • ทำงานใหม่ให้แตก branch เช่น feature/board
  • เขียนข้อความ commit สั้นชัดว่าทำอะไร
  • เปิด Pull Request ให้เพื่อนช่วยรีวิวก่อน merge

✍️ การตั้งชื่อ

  • ตัวแปร/ฟังก์ชัน: ภาษาอังกฤษ camelCase เช่น jobList
  • ชื่อ component: ขึ้นต้นตัวใหญ่ เช่น JobCard
  • ตารางฐานข้อมูล: ตัวเล็ก snake_case เช่น production_records
  • ชื่อให้สื่อความหมาย อ่านแล้วเข้าใจ

✅ เช็กลิสต์ก่อนส่งงานแต่ละเฟส

  • รันในเครื่อง (npm run dev) แล้วใช้งานได้จริง ไม่มี error
  • ลองบนมือถือ (ย่อจอ) ว่าหน้าตายังใช้ได้
  • push ขึ้น GitHub แล้ว Vercel deploy ผ่าน
  • ไม่มีกุญแจลับ (.env.local) หลุดขึ้น GitHub

📖 คำศัพท์ที่เจอบ่อย (อภิธานศัพท์)

คำความหมายแบบเข้าใจง่าย
Frontendส่วนหน้าเว็บที่ผู้ใช้เห็นและกด
Backend / APIส่วนเบื้องหลังที่รับส่งและประมวลผลข้อมูล
Databaseที่เก็บข้อมูลทั้งหมดอย่างเป็นระเบียบ
Deployการเอาเว็บขึ้นออนไลน์ให้คนเข้าใช้ได้
Componentชิ้นส่วน UI ที่เขียนครั้งเดียวใช้ซ้ำได้ เช่น ปุ่ม การ์ด
RLSกฎกำหนดว่าใครเห็น/แก้ข้อมูลแถวไหนได้บ้าง (ความปลอดภัย)
Realtimeข้อมูลอัปเดตให้ทุกคนเห็นทันทีโดยไม่ต้องรีเฟรช
Repository (Repo)ที่เก็บโค้ดของโปรเจคบน GitHub
