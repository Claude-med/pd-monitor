## 1) Flow ครบวงจร (เส้นทางหลัก) — ส่งงานต่อกันทีละแผนก
ทดสอบ 1 งานตั้งแต่เกิดจนเข้าคลัง — แต่ละขั้นทำโดยแผนกที่กำหนด

- [ ] **[ผู้บริหาร]** บอร์ดงาน → "＋ สร้างงานใหม่" → กรอกครบ (ลองกด "＋ เพิ่มยาใหม่" ด้วย) → **สร้างงานผลิต**
      → เด้งเข้าหน้างาน + งานอยู่สถานะ **รอแจ้งผลิต**
- [ ] **[ผู้บริหาร]** ในหน้างาน กด **"ยืนยันแผนผลิต"** → สถานะเป็น **มีแผนแล้ว**
- [ ] **[ผลิต]** เปิดงานเดิม กด **"เริ่มผลิต"** → สถานะเป็น **กำลังผลิต**
- [ ] **[ผลิต]** กด "＋ บันทึกผลผลิต" → กรอกสถานี **เตรียมวัตถุดิบ** (ตั้งต้น/ผลิตได้/ของเสีย/ชม.) → **บันทึก** → ขึ้นในตาราง
- [ ] **[ผลิต]** บันทึกเพิ่มอีก 1–2 สถานี (ผสม → ตอกเม็ด → บรรจุ) ได้ตามจริง
- [ ] **[ผลิต]** กด **"ส่งตรวจ QC"** → สถานะเป็น **QC**
- [ ] **[QC]** เปิดงาน กด **"QC ผ่าน → ส่ง QA"** → ใส่ **รหัสผ่าน** ยืนยันลงนาม → สถานะเป็น **QA** + เห็นลายเซ็น QC
- [ ] **[QA]** เปิดงาน กด **"QA ปล่อยผ่าน → FG"** → ใส่ **รหัสผ่าน** → สถานะเป็น **FG (เข้าคลัง)** + เห็นลายเซ็น QA
- [ ] **[ทุกคน]** บอร์ดงาน: งานนี้ย้ายไปคอลัมน์ **FG** เรียบร้อย

---

## 2) ทดสอบการกันสิทธิ์ / กันทำผิดขั้น (สำคัญ — ความถูกต้อง)
- [ ] **[ผลิต]** ไม่เห็นปุ่ม **"＋ สร้างงานใหม่"** บนบอร์ด (เฉพาะผู้บริหาร) · เข้า `/board/new` ตรงๆ ขึ้น "ไม่มีสิทธิ์"
- [ ] **[ผลิต]** เปิดงานสถานะ **รอแจ้งผลิต** → ไม่มีปุ่ม "เริ่มผลิต" (ต้องให้ผู้บริหารยืนยันแผนก่อน — กันข้ามขั้น)
- [ ] **[QC]** เปิดงานที่ยัง **กำลังผลิต** → ไม่มีปุ่มอนุมัติ QC (ยังไม่ถึงคิวตัวเอง)
- [ ] **[คลัง/warehouse]** เปิดงานใดๆ → ไม่มีปุ่มเปลี่ยนสถานะ (คลังดูได้อย่างเดียว)
- [ ] **[ผลิต]** บันทึกผลผลิตลองกรอก **ผลิตได้ > ตั้งต้น** หรือ **ค่าติดลบ** → ระบบปฏิเสธ + ขึ้นเตือน
- [ ] **[QC]** ลงนามใส่ **รหัสผ่านผิด** → ลงนามไม่สำเร็จ (กันคนอื่นมากดแทน)

---

## 3) ตีกลับ (Reject) — QC/QA ไม่ผ่าน
- [ ] **[QC]** งานสถานะ QC กด **"QC ตีกลับ"** → ต้องใส่ **เหตุผล** + รหัสผ่าน → สถานะกลับเป็น **กำลังผลิต** + เห็นลายเซ็น (แดง) + เหตุผล
- [ ] **[ผลิต]** เห็นงานกลับมาที่ตัวเอง แก้แล้วส่ง QC ใหม่ได้
- [ ] **[QA]** ทดสอบ "QA ตีกลับ" แบบเดียวกัน (กลับไปกำลังผลิต)

---

## 4) ฟีเจอร์เด่น (เปิดใช้จริงต้องเวิร์ก)
- [ ] **อัปเดตสด (Realtime):** เปิดบอร์ด 2 จอ → จอ A เปลี่ยนสถานะงาน → จอ B ขยับเองใน 1–3 วิ (ไม่ต้องรีเฟรช) + เห็นป้าย "🟢 อัปเดตสด"
- [ ] **บันทึกทนเน็ตหลุด:** ระหว่างบันทึกผลผลิต ปิดเน็ต (มือถือ: เปิด Airplane) → กดบันทึก → ขึ้น "กำลังลองใหม่/ค้างไว้" → เปิดเน็ตกลับ → บันทึกเอง **ไม่มีแถวซ้ำ**
- [ ] **มือถือ:** เปิดจากมือถือจริง → แตะช่องกรอก **ไม่ซูมเอง** · ปุ่มกดง่าย · เมนูเปิด/ปิดลื่น
- [ ] **รายงานประจำวัน:** เมนู "รายงานประจำวัน" → เลือกวันที่ → เห็นบันทึกผลผลิตของงานที่ทดสอบ + ยอดรวมถูก
- [ ] **แดชบอร์ด (ผู้บริหาร):** หน้าแรก → ตัวเลขงานตามสถานะ + ผลผลิต/Yield ตรง · เห็นส่วน **ต้นทุนค่าแรง** (เฉพาะผู้บริหาร)
- [ ] **ประวัติ (audit):** ตรวจว่าทุกการเปลี่ยนสถานะ/บันทึก/ลงนาม มีเก็บ "ใครทำ–เมื่อไหร่" (ผู้บริหาร/QA ตรวจใน Supabase ตาราง `audit_log` ได้)

---

## 5) ความพร้อมเปิดใช้จริง (go-live)
- [ ] สร้างบัญชีผู้ใช้จริงครบทุกคน (~30 คน) + กำหนด role ให้ถูกแผนก
- [ ] ลบ/กันข้อมูลตัวอย่าง (seed) ไม่ให้ปนกับงานจริง (หรือเริ่มเลขงานจริงแยกชุด)
- [ ] อธิบายผู้ใช้: ลงชื่อเข้าใช้ยังไง · ทำอะไรได้ตามแผนก · ลงนามใช้รหัสผ่านตัวเอง
- [ ] ผู้บริหารรู้วิธีสร้างงานใหม่ + ดูแดชบอร์ด
